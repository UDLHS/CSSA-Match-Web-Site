CSSA Cricket Fiesta '26 — Platform UI/UX Spec

Public live-score site + admin scoring console · Next.js App Router · Tailwind CSS + shadcn/ui · mobile-first, light by default with full dark tokens. Visuals: see Design Explorations (palette switchable via Tweaks).

A · Overall UI concept

Premium tournament feel built on three moves: (1) a deep-navy "ink" gradient hero that owns every live-score moment, with the largest type on the page reserved for score digits; (2) white cards on a cool light-gray field with one consistent radius (16px cards / 10px controls) and soft two-layer shadows; (3) color discipline — team colors appear only in logos/monograms and avatars, gold appears only for ranks, player-of-the-match and free-hit, red is reserved for LIVE and wickets. The admin console uses the same tokens but dark ink chrome (sidebar) so a scorer can never confuse it with the public site.

C · Typography

  • Display / headings / scores: Barlow Condensed 600–700 (fallback: "Arial Narrow", sans-serif). Sporty, tall, space-efficient on mobile.
  • Body / UI / tables: IBM Plex Sans 400–600 (fallback: system-ui).
  • Scale: score-xl 64 · score-lg 40 · score-md 26 · h1 36 · h2 26 · h3 20 · body 14.5 · small 13 · label 11 caps +9% tracking.
  • Tabular numerics: every live-updating number gets font-variant-numeric: tabular-nums (IBM Plex Sans supports tnum) and a reserved digit width (min-width in ch) so 99/2 → 100/2 causes zero layout shift. Hero scores in Barlow Condensed sit in a fixed-width container for the same reason.

D · Public pages

Home

  • Header: logo + "CRICKET FIESTA '26 / CSSA · University of Kelaniya" lockup, nav (Home, Matches, Players, Leaderboard, Popular), theme toggle, ghost Admin button. Mobile: logo + theme + hamburger → shadcn Sheet (left).
  • LiveMatchHero: ink-gradient card. Desktop: 3-col grid (team A score · VS · team B score, batting side full-opacity, bowled-out side at 82%), stats strip (Target / Need / CRR / RRR) in a translucent inset, bottom row = striker•/non-striker/bowler with figures + last-6-balls strip + gold "Full scorecard" button. Mobile: stacked teams, 4-col stat grid, players stack, balls + CTA share a row. Toss line small and muted.
  • Role indicator (new): every player avatar carries a small corner badge reflecting current activity — a bat glyph (primary blue) while batting, a ball glyph (accent) while bowling, and no badge / name only while fielding. The same rule drives the hero's striker/non-striker/bowler discs and the leaderboard (bat on the Batting tab, ball on Bowling).
  • Match cards (now a swipeable carousel): three snap slides — Previous ← Live → Upcoming — opening centered on Live. Swipe (touch) or arrow buttons (desktop) move between them; a segmented control + dot indicator stay in sync. Each slide is a 3-up grid (1-col mobile) of the cards described below.
  • Sponsor slots (new): a leaderboard banner sits between the LiveMatchHero and the matches carousel; a skyscraper sits between the leaderboard preview and popular players (becomes a banner when the columns stack on mobile). Both are placeholders fed by a sponsors/ad table; Player-of-the-Match carries a “presented by” tag and the footer adds a partners strip.
  • Leaderboard preview: card with pill tabs Batting/Bowling/Overall, top-5 rows (rank — gold for #1 — avatar, name, team chip, headline stat), full-width soft "View full leaderboard" button.
  • Popular players: "★ Official popularity ranking" label; top 3 enlarged (rank disc gold/silver/bronze), rest 4-up grid → 2-up mobile. Card: photo, name, team chip, role badge, vote count.
  • Footer: ink bar, lockup + copyright.

Match details modal

Desktop: centered Dialog 860px, ink header (badge, both scores, need line, close), underline Tabs: Summary · Scorecard · Fall of Wickets · Ball-by-Ball · Info. Mobile: full-screen Sheet, header compresses, tab bar sticky under it. Scorecard = batting table (dismissal under the name on mobile), extras/total strip, bowling table, FoW teaser line. FoW tab = 7-col table (wkt, score, over, player, type, bowler, fielder/helper). Ball-by-ball = over sections with colored ball dots + one-line commentary.

Leaderboard page

Title, search input, team Select, three tabs. Desktop = full Table (Batting: Rank Player Team M Runs Balls SR Avg 4s 6s HS; Bowling: O R W Econ Avg SR Best; Overall: Bat/Bowl/Field/Total pts). Mobile collapse rule: card keeps Rank + avatar + Player + team chip + headline stat (Runs/Wkts/Total) + chevron; all remaining columns move into an expandable details grid inside the card.

Player modal

Ink header: large photo, name, team chip, batting/bowling styles, role badge, vote badge. Tabs Overview / Batting / Bowling / Recent Matches; Overview = 3×2 stat cards (runs, SR, avg, wickets, HS, votes).

E · Admin area

Layout: 220px ink sidebar (Dashboard, Teams, Players, Matches, Venues, Live Scoring — red dot when live, Popular Votes, Sponsors & Ads, Leaderboard, Admin Users, Audit Logs, Settings; active = gold inset bar), topbar (page title, theme, avatar + role, logout). Mobile: sidebar becomes a Sheet drawer from the hamburger.

Management pattern — every entity is admin-editable via a consistent list → edit flow (searchable/filterable table → full form):

  • Teams: list (logo, short, colors, squad size, P/W/L, status) → editor: name, short (≤4), logo upload, primary/secondary color (contrast-checked), home venue select, captain, coach, group, founded, status, bio, and an inline squad table (add/remove/reorder players).
  • Players: filterable list (team, role, status) → editor: photo, name, team, jersey, DOB, status, primary role, batting hand, bowling style, captain toggle, bio, and an admin vote override with audit note (stats are computed, read-only).
  • Matches: list (#, teams, date, venue, format, status) → editor in sections — Fixture (stage, teams, number), Schedule & venue (date, time, venue select + add-venue inline, format preset T20/T10/ODI/Hundred/Custom, overs, balls/over), Toss & officials, and a two-column Playing XI checklist (11/side).
  • Venues: dedicated manager — the admin defines every place (name, location, capacity, pitch type, availability) that the match form can select.
  • Sponsors & Ads: sponsors CRUD (name, tier, website, status) + an ad-placement board (each public slot with current creative, schedule, impressions, live toggle) + a “Load new ad” form where the admin picks a placement (auto-fills required dimensions), assigns a sponsor, uploads the creative, sets click-URL, schedule and rotation weight.
  • Popular votes: per-player count steppers + override, voting window open/close, one-vote-per-device and public-visibility rules.
  • Leaderboard: rebuild engine — status, last-rebuilt, one-click recompute of batting/bowling/points/standings/popularity.
  • Admin users: users list with role (Super admin/Editor/Scorer/Viewer), scope, status, plus a configurable role-permission matrix.
  • Audit logs: full who/what/when table with user/action/date filters + CSV export.
  • Settings: tournament name/season/logo, default palette & mode, format defaults, points system, voting rules, and a danger zone (reset data).

Dashboard: 4 stat cards; a live-match card with red border + "Open scoring console" CTA; today's fixtures + recent audit log side by side.

Live scoring console (most important screen)

  • A — summary strip (ink): score, overs, target, need, CRR, RRR, innings, FREE HIT badge when armed.
  • B — current players: striker highlighted (primary-soft fill + dot), non-striker, bowler; each row has a Change button; "⇄ Swap ends" in the panel header.
  • C — quick pad: 3×2 grid of 0 1 2 3 4 6 (64px tall, 4 in blue, 6 in violet), extras row (Wide/No ball/Bye/Leg bye, amber), WICKET (red, 2-span) + Undo. A run or dot is one tap, no confirm. On free hit the pad shows "⚡ FREE HIT — dismissals limited to run-out" and the WICKET button dims; its dialog pre-filters to run-out only.
  • D — wicket dialog (mobile: bottom Sheet): dismissal-type chip grid, player out, "bowler credited" toggle, fielder/thrower/keeper, direct-hit toggle, end, new batter, notes → red Confirm.
  • E — last balls: last 6 with over no., dot, description; Undo (ghost, latest only) is visually distinct from Edit (dashed primary — triggers recompute, always confirms); "View audit history" link.
  • F — match controls: Start/End innings, Complete match, PoM; every "…" button opens a confirm AlertDialog; Complete match is the only red one.
  • Mobile order (390): summary → players → last 3 balls → pad (thumb zone) → controls. All pad targets ≥56px.

Forms & validation

Team (name, short ≤4 chars, logo upload w/ monogram fallback, primary/secondary color with automatic 4.5:1 contrast check), Player (name, photo, team, role, styles, jersey #), Match (tournament, teams, venue, datetime, format, overs, toss, playing-XI selector with per-team checklists + count chip), Votes (search, current count, ± steppers, save → Toast). Inline errors under the field in danger red, never only color: icon + message. Submit disabled until valid.

H · Modal behavior

Desktop Dialog (radius 16, scrim rgba(10,15,35,.55), scale+fade 150ms) ⟷ mobile Sheet (full-screen for match/player details, bottom for wicket/confirm; drag-handle, slide-up 200ms). Tabs stay sticky inside sheets. Esc/scrim closes non-destructive modals only; confirm dialogs require explicit action.

I · Empty / loading / error

Every data view ships all three (drawn in Explorations §A): empty = icon disc + one sentence + next action; loading = skeletons matching final row geometry (never spinners for scores/tables); error = red-tinted card + Retry, live views auto-resume polling. Score digits reserve width so updates never shift layout.

J · Accessibility

  • Team colors used behind text are contrast-checked at save time (Team form blocks <4.5:1).
  • Status is never color-only: badges carry text, wickets carry "W", free hit has label + icon.
  • aria-live="polite" on hero score + last-balls strip; ball dots get aria-label ("over 16.2: six").
  • Scoring pad: real buttons, focus rings, arrow-key roving focus, 0–6 as keyboard shortcuts; modals trap focus and return it.
  • LIVE pulse and entrance animations respect prefers-reduced-motion.
